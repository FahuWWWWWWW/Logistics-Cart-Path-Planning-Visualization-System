# 物流小车路径规划系统 - 离线启动工具

## 启动方式（任选其一）

### 方式一：一键静默启动（推荐）
**文件：** `一键启动.vbs`

- 双击即启动，**全程无弹窗**
- 自动在后台启动服务器，等待就绪后打开浏览器
- 最适合日常使用

> 启动后浏览器会自动打开，任务栏会出现一个 Node.js 后台进程。
> 如需停止服务器，双击 `停止.vbs` 或直接在任务管理器结束 node.exe。

---

### 方式二：图形化启动器
**文件：** `启动器.hta`

- 提供可视化操作界面，可查看服务器状态、启动日志
- 支持手动启动/停止服务器、打开浏览器
- 适合需要查看启动过程的用户

> 注意：HTA 使用 Windows 内置的 IE 渲染引擎，界面效果较浏览器简陋，但功能完整。

---

### 方式三：命令行启动
**文件：** `启动.bat`

- 显示启动过程的命令行输出
- 关闭命令行窗口即停止服务器
- 适合调试或排查问题

---

### 停止服务器
**文件：** `停止.vbs`

- 双击即可停止后台运行的 Node.js 服务器

---

## 常见问题

### Q: 提示 "dist 文件夹不存在"
**原因：** 项目还没有执行过构建。  
**解决：** 在项目根目录运行 `npm run build`，生成 `dist` 文件夹后再使用启动工具。

### Q: 提示 "服务器启动失败 / Node.js 未安装"
**原因：** 系统没有安装 Node.js，或环境变量配置不正确。  
**解决：**
1. 下载安装 Node.js：https://nodejs.org/
2. 安装完成后，打开命令提示符，输入 `node --version` 验证

### Q: 端口 8899 被占用
**解决：**
- 方式一：先运行 `停止.vbs` 释放端口
- 方式二：修改 `server.cjs` 中的 `PORT` 常量，使用其他端口

### Q: HTA 启动器界面显示异常
**原因：** Windows 部分版本 HTA 使用旧版 IE 引擎渲染，现代 CSS 特性不支持。  
**解决：** 使用 `一键启动.vbs`，体验更好。
