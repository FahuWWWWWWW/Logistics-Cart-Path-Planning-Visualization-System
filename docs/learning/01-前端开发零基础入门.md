## 1. 什么是前端开发？

### 1.1 简单理解

**前端开发** = 用户能看到和交互的部分

想象一个餐厅：
- **前端** = 餐厅的装修、菜单、服务员（顾客能看到的）
- **后端** = 厨房（顾客看不到的，但负责做菜）

### 1.2 前端工程师做什么？

- 设计网页布局和外观
- 实现用户交互功能（点击、输入、动画等）
- 确保网页在不同设备上正常显示（手机、平板、电脑）

### 1.3 本项目中的前端

我们的项目是一个**物流小车路径规划可视化系统**，前端部分包括：
- 显示 48×48 的网格地图
- 显示小车位置和路径
- 提供按钮和控件供用户操作
- 通过串口与下位机通信

## 2. 前端三大核心技术

前端开发必须掌握三大技术：**HTML**、**CSS**、**JavaScript**

| 技术 | 作用 | 类比 |
|------|------|------|
| **HTML** | 网页结构（骨架） | 人的骨骼 |
| **CSS** | 网页样式（外观） | 人的皮肤、衣服 |
| **JavaScript** | 网页交互（行为） | 人的肌肉、神经 |

### 2.1 HTML（超文本标记语言）

**作用**：定义网页的结构和内容

**示例**：
```html
<!DOCTYPE html>
<html>
<head>
    <title>我的第一个网页</title>
</head>
<body>
    <h1>这是一个标题</h1>
    <p>这是一个段落</p>
    <button>这是一个按钮</button>
</body>
</html>
```

### 2.2 CSS（层叠样式表）

**作用**：美化网页，控制布局、颜色、字体等

**示例**：
```css
/* 设置标题样式 */
h1 {
    color: blue;           /* 文字颜色：蓝色 */
    font-size: 24px;       /* 字体大小：24像素 */
    text-align: center;    /* 居中对齐 */
}

/* 设置按钮样式 */
button {
    background: green;     /* 背景色：绿色 */
    color: white;          /* 文字颜色：白色 */
    padding: 10px 20px;    /* 内边距：上下10px，左右20px */
    border: none;          /* 无边框 */
    border-radius: 5px;    /* 圆角：5像素 */
}
```

### 2.3 JavaScript（JS）

**作用**：实现网页的交互功能

**示例**：
```javascript
// 点击按钮时弹出提示
document.querySelector('button').addEventListener('click', function() {
    alert('你好，世界！');
});

// 更简单的写法（推荐）
document.querySelector('button').onclick = function() {
    alert('你好，世界！');
};
```

## 3. 开发环境搭建

### 3.1 安装 Node.js

Node.js 是一个 JavaScript 运行环境，前端开发必备。

**步骤**：
1. 访问 https://nodejs.org/
2. 下载并安装 LTS（长期支持）版本
3. 安装完成后，打开命令提示符（CMD），输入：
   ```bash
   node --version
   npm --version
   ```
   如果显示版本号，说明安装成功。

### 3.2 安装代码编辑器

推荐使用 **Visual Studio Code（VS Code）**

**步骤**：
1. 访问 https://code.visualstudio.com/
2. 下载并安装
3. 安装以下插件（在 VS Code 的扩展商店搜索）：
   - Chinese (Simplified) Language Pack（中文语言包）
   - Live Server（实时预览）
   - Prettier（代码格式化）

### 3.3 克隆本项目

```bash
# 克隆项目到本地
git clone https://github.com/FahuWWWWWWW/Logistics-Cart-Path-Planning-Visualization-System.git

# 进入项目目录
cd Logistics-Cart-Path-Planning-Visualization-System

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

## 4. HTML 基础

### 4.1 HTML 文档结构

```html
<!DOCTYPE html>
<html lang="zh-CN">

    <title>网页标题</title>
```

### 4.2 常用 HTML 标签

| 标签 | 含义 | 示例 |
|------|------|------|
| `<h1>` ~ `<h6>` | 标题（1级到6级） | `<h1>主标题</h1>` |
| `<p>` | 段落 | `<p>这是一段文字</p>` |
| `<div>` | 容器（块级） | `<div>内容</div>` |
| `<span>` | 容器（行内） | `<span>文字</span>` |
| `<button>` | 按钮 | `<button>点击我</button>` |
| `<input>` | 输入框 | `<input type="text">` |
| `<img>` | 图片 | `<img src="图片地址">` |
| `<a>` | 链接 | `<a href="网址">文字</a>` |

### 4.3 实际项目中的 HTML

查看本项目的 `index.html`：

```html
<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>物流小车路径规划可视化系统</title>
  </head>
```

**关键点**：
- `<div id="root">`：React 应用的挂载点
- `<script type="module" src="/src/main.tsx">`：引入 React 入口文件

## 5. CSS 基础

### 5.1 CSS 的三种引入方式

**方式一：内联样式**（不推荐）
```html
<div style="color: red;">红色文字</div>
```

**方式二：内部样式表**
```html
<head>
    <style>
        div {
            color: red;
        }
    </style>
</head>
```

**方式三：外部样式表**（推荐）
```html
<head>
    <link rel="stylesheet" href="style.css">
</head>
```

### 5.2 CSS 选择器

| 选择器 | 含义 | 示例 |
|--------|------|------|
| `*` | 通配符（所有元素） | `* { margin: 0; }` |
| `div` | 标签选择器 | `div { color: red; }` |
| `.class` | 类选择器 | `.button { ... }` |
| `#id` | ID 选择器 | `#root { ... }` |

### 5.3 Flexbox 布局（重点）

Flexbox 是现代网页布局的核心技术。

**示例**：
```css
/* 容器设置为 flex 布局 */
.container {
    display: flex;
    justify-content: center;   /* 水平居中 */
    align-items: center;       /* 垂直居中 */
    flex-direction: column;    /* 垂直排列 */
}

/* 子元素 */
.item {
    flex: 1;   /* 占满剩余空间 */
}
```

### 5.4 实际项目中的 CSS

本项目使用 **Tailwind CSS**（ utility-first CSS 框架），通过类名直接写样式：

```html
<!-- 传统 CSS -->
<div style="display: flex; justify-content: center;">
    <button style="background: blue; color: white;">按钮</button>
</div>

<!-- Tailwind CSS（本项目使用） -->
<div class="flex justify-center">
    <button class="bg-blue-500 text-white">按钮</button>
</div>
```

**优势**：无需写 CSS 文件，直接在 HTML 中写样式类名。

## 6. JavaScript 基础

### 6.1 变量声明

```javascript
// 旧方式（不推荐）
var name = '张三';

// 新方式（推荐）
let age = 25;        // 可以修改
const PI = 3.14;     // 不能修改
```

### 6.2 函数

```javascript
// 传统函数
function add(a, b) {
    return a + b;
}

// 箭头函数（推荐）
const add = (a, b) => {
    return a + b;
};

// 简化写法
const add = (a, b) => a + b;
```

### 6.3 对象

```javascript
// 定义对象
const person = {
    name: '张三',
    age: 25,
    sayHello: function() {
        console.log('你好，我是' + this.name);
    }
};

// 访问属性
console.log(person.name);   // 输出：张三
person.sayHello();           // 输出：你好，我是张三
```

### 6.4 数组

```javascript
// 定义数组
const colors = ['红色', '绿色', '蓝色'];

// 访问元素
console.log(colors[0]);   // 输出：红色

// 遍历数组
colors.forEach(function(color) {
    console.log(color);
});
```

### 6.5 异步编程（重点）

JavaScript 是单线程的，异步编程很重要。

**回调函数**（旧方式）
```javascript
setTimeout(function() {
    console.log('2秒后执行');
}, 2000);
```

**Promise**（新方式）
```javascript
fetch('https://api.example.com/data')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error(error));
```

**async/await**（最新方式，推荐）
```javascript
async function getData() {
    try {
        const response = await fetch('https://api.example.com/data');
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error(error);
    }
}
```

## 7. React 框架入门

### 7.1 什么是 React？

React 是 Facebook 开发的**前端框架**，用于构建用户界面。

**核心概念**：
- **组件**：可复用的 UI 单元
- **状态（State）**：组件的数据
- **属性（Props）**：组件之间传递数据

### 7.2 第一个 React 组件

```jsx
// 定义一个组件
function Welcome() {
    return <h1>你好，世界！</h1>;
}

// 使用组件
<Welcome />
```

### 7.3 状态（State）示例

```jsx
import { useState } from 'react';

function Counter() {
    // 声明状态变量 count，初始值为 0
    const [count, setCount] = useState(0);
    
    return (
        <div>
            <p>当前计数：{count}</p>
            <button onClick={() => setCount(count + 1)}>
                点击 +1
            </button>
        </div>
    );
}
```

### 7.4 实际项目中的 React

查看本项目的 `src/App.tsx`：

```tsx
function App() {
    const [theme, setTheme] = useState<'light' | 'dark'>('dark');
    
    return (
        <div className={theme === 'dark' ? 'dark' : ''}>
            <GridMap />       {/* 地图组件 */}
            <ControlPanel />  {/* 控制面板组件 */}
        </div>
    );
}
```

## 8. TypeScript 入门

### 8.1 什么是 TypeScript？

TypeScript 是 JavaScript 的**超集**，添加了**类型系统**。

**优势**：
- 提前发现错误
- 代码提示更友好
- 适合大型项目

### 8.2 基础类型

```typescript
// 基本类型
let name: string = '张三';
let age: number = 25;
let isStudent: boolean = true;

// 数组
let scores: number[] = [90, 85, 95];

// 对象
let person: {
    name: string;
    age: number;
} = {
    name: '张三',
    age: 25
};

// 函数
function add(a: number, b: number): number {
    return a + b;
}
```

### 8.3 接口（Interface）

```typescript
// 定义接口
interface Person {
    name: string;
    age: number;
    email?: string;   // 可选属性
}

// 使用接口
const student: Person = {
    name: '李四',
    age: 22
};
```

## 9. 项目实战：理解我们的项目

### 9.1 项目结构

```
path-planning-visualization/
├── src/                   # 源代码
│   ├── components/        # React 组件
│   │   ├── GridMap.tsx    # 地图组件
│   │   ├── ControlPanel.tsx  # 控制面板组件
│   │   └── SerialPanel.tsx   # 串口面板组件
│   ├── utils/             # 工具函数
│   │   ├── serial.ts      # 串口通信
│   │   └── pathPlanning.ts  # 路径规划算法
│   ├── App.tsx            # 主应用组件
│   └── main.tsx           # 入口文件
├── public/                # 静态资源
├── index.html             # HTML 模板
├── package.json           # 项目配置
└── vite.config.ts         # Vite 配置
```

### 9.2 启动流程

```
1. 用户打开浏览器访问 http://localhost:5173
2. 浏览器加载 index.html
3. index.html 引入 src/main.tsx
4. main.tsx 渲染 <App /> 组件
5. App 组件加载 GridMap 和 ControlPanel
6. 用户通过 ControlPanel 操作，GridMap 显示结果
```

### 9.3 核心功能解析

**功能一：显示地图**
- `GridMap.tsx` 使用 Canvas API 绘制 48×48 网格
- 每个格子代表 50mm × 50mm

**功能二：路径规划**
- `utils/pathPlanning.ts` 实现 A* 算法
- 用户输入起点和终点，计算出最优路径

**功能三：串口通信**
- `utils/serial.ts` 使用 Web Serial API
- 与下位机（K230/STM32）通信
